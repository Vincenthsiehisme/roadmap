      能力定位

      成熟度路徑

      RAG Response → Context Routing → Tool Calling → Guardrails / Human Handoff → Runtime Monitoring

      目前採用方式

      先以固定旅遊情境、知識節點、工具調用與回答依據 trace 落地，再逐步導入個人化與更高自動化。

      方法池 / 任務採用關係
      RAGKnowledge RetrievalContext RoutingIntent ClassificationTool CallingAgent WorkflowLLM EvaluationAnswer GroundingGuardrailsFeedback ClassificationSentiment ClassificationTrace ReviewHuman HandoffRuntime MonitoringJourney Stage Context
      輸入資料

      旅程階段、商品與訂單資訊、FAQ / 知識庫、搜尋與推薦結果、使用者上下文、服務流程與工具權限。

      輸出物件

      知識節點回覆、工具調用結果、行前 / 行中 / 行後互動、異常轉人工、prompt trace 與品質評估。

      能力演進
      初期固定情境腳本與知識節點
      中期Agent Workflow + Tool Calling + Trace
      後期個人化旅程互動與跨服務調用
      初版驗證
      • AI 回覆是否能被 trace 與評估
      • 行前、行中、行後互動是否產生有效行為
      • 工具調用成功率、成本、延遲與異常率
      階段邊界AI 互動層聚焦情境承接、知識組織、工具調用與回覆品質觀測，並銜接搜尋推薦能力。
