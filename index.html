Phase 1

資料基礎與事件觀測

D資料準備O成效觀測

作為搜尋、推薦、活化、投放與 AI 互動服務的共同基礎階段。

初版定義
完成事件規範、關鍵行為補點與基本成效回收，不在此文件展開欄位、表設計或 API payload。

Data Foundation / Measurement

能力定位

建立事件規範、漏斗觀測、ID 關聯與回饋資料，作為搜尋、推薦、活化、投放與 AI 互動的共同基礎。

成熟度路徑

Event Taxonomy → Funnel Measurement → Identity Resolution → Feedback Loop

目前採用方式

先確認 Amplitude 事件品質、關鍵行為補點與基本成效回收；資料表、API 與權限細節由治理文件承接。

Phase 3

AI 搜尋機制

S搜尋理解H混合召回

定位為搜尋理解與排序能力，並與 AI Agent、對話服務形成清楚分工。

初版定義
先以 Query Understanding、Keyword Retrieval、Vector Retrieval、Filter 與 Reranking 的小流量驗證為主。

Query Understanding / Hybrid Search

能力定位

讓搜尋能理解自然語言需求，並結合關鍵字、向量召回、條件過濾與排序機制，提高搜尋結果相關性。

成熟度路徑

Keyword Retrieval → Query Understanding → Vector Retrieval → Hybrid Search / Reranking

目前採用方式

先以小流量驗證查詢改寫、意圖辨識、keyword retrieval、vector retrieval、filter 與 reranking。

query、目的地與 POI、天數、月份、價格帶、商品可售狀態、搜尋曝光、點擊、改搜與查無結果。

輸出物件

query intent、改寫後查詢、候選商品池、facet 條件、排序分數、zero-result 原因與搜尋品質回饋。

  • 改搜率、搜尋後 booking_start 率
階段邊界AI 搜尋屬搜尋理解與排序能力；AI 互動服務或 Agent Workflow 由 AI 互動服務層承接。

Phase 2 → 4

推薦機制

R推薦能力B批次產出I意圖濃度

由規則、批次與相似商品先行，逐步走向 session-based 與意圖驅動推薦。

初版定義
初期以 batch / session-based 驗證商品池、排序與回饋，後續逐步銜接 full real-time 能力。

Recommendation / Personalization

能力定位

由熱門、相似商品與使用者行為訊號，逐步建立個人化與情境化推薦能力。

成熟度路徑

Popularity → Similarity → Segment-based → Session-based → Intent-driven Recommendation

目前採用方式

初期以熱門商品、相似商品與手動分眾推薦驗證商品池與排序品質，再銜接 session-based 與 near real-time features。

輸入資料

商品屬性、地區與主題標籤、曝光、點擊、瀏覽深度、收藏、booking_start、成交、近期熱度與外部趨勢。

輸出物件

熱門分數、趨勢分數、相似商品清單、分眾推薦商品池、session 推薦結果與排序理由。

  • 推薦商品進入詳情頁、收藏、booking_start 的增量
  • 熱門分數是否排除高曝光污染
階段邊界即時推薦屬後續成熟能力；初期先以 batch / session-based 能力驗證商品池、排序與回饋，再逐步銜接 event streaming、online feature 與低延遲服務。

成熟度路徑

Association Rule → Lifecycle Stage → Next Best Offer → Next Best Action

目前採用方式

先以商品關聯規則與加購場景驗證，再逐步導入旅程階段、可售性、頻控與服務負荷控管。

輸出物件

階段邊界交叉銷售會依旅程階段、可售狀態、頻控、服務負荷與同意狀態進行控管。
